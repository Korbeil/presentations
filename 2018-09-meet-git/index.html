1. What is git ?

Git basics

  • Working: You modify files in your working tree.
  • Staging: You selectively stage just those changes you want to be part of your next commit.
  • Repository: You do a commit.

Your local git is a repository

Distant git is a repository (and you can have multiple distant git, for example when using forks)

Each repository have their own branches, you won't recover them by simply fetching a repository

References

A lot in theses !

HEAD

You always have a HEAD when on a branch

It contains a reference to the last commit of that branch

Tag

Points to a commit with some details:

name, author, date, description

Remote

It store, for each branch, the last commit you pushed to them.

2. Usage


Logs !

Pretty logs ?

Quite easy:

git log --pretty=format:"%h %s" --graph

Usefull logs !

git reflog is decoupled from HEAD and repository. It's just a local history of git.
